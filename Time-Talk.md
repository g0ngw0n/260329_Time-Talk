### 1-1. 프로젝트 목적

사용자가 과거의 특정 날짜에 메시지를 “보낸 것처럼” 기록하고,  
해당 날짜의 UI에서 실제로 메시지가 도착한 것처럼 표시하여  
감성적인 시간 기록 경험을 제공하는 것을 목표로 한다.

### 1-2. 대상 사용자

- 감성 기록을 좋아하는 사용자
- 일기 / 회고를 색다르게 남기고 싶은 사용자

## 2. 기능 요구사항

### 2-1. 기능 목록

---

### 2-2. 기능 상세

#### F-01: 메시지 작성

- 설명: 날짜를 선택하고 메시지를 입력할 수 있다
- 사용자 시나리오: 날짜 선택 후 텍스트 입력 시 작성 폼이 표시된다
- 입력: 날짜, 텍스트
- 동작: 입력된 메시지가 저장 대기 상태가 된다
- 예외 처리:
  - 날짜 미선택 시 저장 불가
  - 텍스트 공백 시 경고 메시지 표시

---

#### F-02: 메시지 저장

- 설명: 메시지를 브라우저에 저장한다
- 사용자 시나리오: 저장 버튼 클릭 시 메시지 저장
- 동작:
  - localStorage에 저장
  - 저장 완료 알림 표시
- 예외 처리:
  - 저장 실패 시 에러 메시지 표시

---

#### F-03: 타임라인 표시

- 설명: 날짜별 메시지를 정렬하여 표시
- 사용자 시나리오: 날짜 선택 시 해당 메시지 출력
- 동작:
  - 로딩 시 Skeleton UI 표시
  - 데이터 없으면 "메시지가 없습니다" 출력

---

#### F-04: 메시지 표시 연출

- 설명: 메시지가 과거에 도착한 것처럼 보이게 UI 연출
- 동작:
  - Fade-in / Slide 애니메이션 적용
  - 채팅 UI 형태로 표시 가능

---

#### F-05: 메시지 수정/삭제

- 설명: 저장된 메시지를 수정 또는 삭제
- 동작:
  - 수정 시 즉시 반영
  - 삭제 시 확인 모달 표시
